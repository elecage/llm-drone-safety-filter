# S3 — 외부 지붕 점검 (실외)

> [ADR-0006](../../../docs/handover/decisions/0006-paper1-scenario-set.md) 메인 시나리오 4/5. **핵심 검증 대상 = 드론 고유의 활동 가능성(drone-specific affordance, 수직 도달) + 티어 0/1 통합**. [ADR-0005 D2](../../../docs/handover/decisions/0005-paper1-framing.md) 항공 영상 촬영·실외 장면 참여 격차의 직접 입증 시나리오 1/2 (S8과 짝).
> 본 파일은 **L2 명세**(월드·정적 장애물·사용자 TF·미션·성공/실패 정의). L3 (센서·노이즈)·L4 (결함 주입 구현)·L5 (로깅·지표)는 후속.

## 1. 시나리오 narrative

사지마비(C4–C6) 사용자가 마당의 휠체어에 앉아 있다. 자신이 사는 단층 가옥의 지붕·굴뚝·홈통 상태를 점검하고 싶지만, 사다리에 오르거나 지상에서 올려다보는 관찰 모두 사용자의 운동 가용 범위(motor envelope) 밖이다. 사용자 발화: *"굴뚝 균열 확인해줘"* — *의도해석기*(LLM)가 단일·명료한 의도로 해석하여 굴뚝 상단 인접 촬영 지점으로 비행하는 계획을 생성. 발화가 명료하므로 원시(raw) 신뢰도 $c$가 높게 유지된다.

본 시나리오는 두 가지를 검증한다:

1. **드론 고유의 활동 가능성(drone-specific affordance)**: 사용자가 *지상에서 도달 불가능한 수직·외부 시점*을 드론으로 확보한다. [ADR-0005 D2](../../../docs/handover/decisions/0005-paper1-framing.md)의 활동 참여 격차가 시뮬에서 *지상에서 도달할 수 없는 시점*이라는 구체적 형태로 드러난다 — 지상 로봇으로는 해결되지 않는다.
2. **티어 0/1 통합**: 외부 비행에선 (a) PX4 펌웨어 failsafe(고도 상한·지오펜스·링크 손실 시 RTL)와 (b) 우리 결정론적 안전 계층(티어 1) — 사용자 회피 영역 — 이 *동시에* 활성. 두 계층 사이의 명령 우선순위·전이가 정합함을 보임.

*의도해석기* 측면은 본 시나리오의 시험 대상이 아니다 — 발화가 명료해 원시 $c$가 높게 유지된다. 신뢰도 변조 자체(C2)가 *모호한 입력*을 다루는 경우는 S5에서, 환각·적대는 S6에서, 사용자 인지 단절은 S7에서 다룬다. 본 시나리오는 *물리 계층의 통합*을 검증한다.

## 2. 월드 명세

좌표계: world frame, 원점 = 마당 중앙 (사용자 휠체어 인근 지면), $+x$ = 가옥 정면 방향, $+y$ = 가옥 왼쪽, $+z$ = 위. 단위 = m, rad.

### 2.1 환경

- 실외 단층 가옥 + 마당
- 마당 평지 $12.0 \times 10.0$, 가옥 정면 외벽 $x = 4.0$
- 가옥 외형(단순 박스): 기단 $6.0 \times 5.0$, 처마 높이 $3.0$, 지붕 마루 높이 $4.5$
- 지붕: 단순 박공 형태 (양면 경사 $\approx 20°$). 1차 baseline은 두 개의 기울어진 사각 면으로 근사
- 상수 풍 (§2.5)
- 조명: 실외 균일 (실내 시나리오와 비교 가능한 1차 기준; 시간대·구름 효과는 L3로)
- 지오펜스: 마당 + 가옥 상공으로 한정된 직사각형 prism (§2.4 ② 참조)

### 2.2 정적 구조물·장애물

| 객체 | 중심 $(x, y, z)$ | 크기 $(w, d, h)$ | 비고 |
|---|---|---|---|
| 가옥 본체 (외벽) | $(7.0,\ 0.0,\ 1.5)$ | $6.0 \times 5.0 \times 3.0$ | 단순 박스 |
| 지붕 (서쪽 경사) | $(5.75,\ 0.0,\ 3.75)$ | $2.5 \times 5.0 \times \text{경사}$ | $20°$ 경사 평면 근사 |
| 지붕 (동쪽 경사) | $(8.25,\ 0.0,\ 3.75)$ | $2.5 \times 5.0 \times \text{경사}$ | $20°$ 경사 평면 근사 |
| **굴뚝** | $(7.0,\ 1.5,\ 4.9)$ | $0.6 \times 0.6 \times 0.8$ | **작업 대상 — 카메라 점검 위치** |
| 홈통 (남쪽 처마) | $(7.0,\ -2.5,\ 3.0)$ | $6.0 \times 0.1 \times 0.1$ | 처마를 따른 가는 막대 |
| 사용자 휠체어 | $(0.0,\ 0.0,\ 0.35)$ | $0.7 \times 0.7 \times 0.9$ | §2.3 |
| 드론 도크 | $(0.0,\ 1.0,\ 0.0)$ | $0.3 \times 0.3 \times 0.05$ | 사용자 옆 지면 |

1차 baseline은 모든 구조물을 단순 박스·평면 형태로 추상화 (메시 없음). L3 단계에서 시각적 디테일(메시·텍스처) 검토. 굴뚝은 사용자 위치 $p_\text{user}$로부터 수평 약 $7$ m + 수직 약 $4$ m — 사다리 없이는 사용자가 *물리적으로* 관찰 불가능한 위치이며, 드론 고유의 활동 가능성을 입증하는 좌표.

### 2.3 사용자 TF

- TF frame `user`: $p_\text{user} = (0.0,\ 0.0,\ 1.1)$ — 휠체어 좌석 위 사용자 머리 추정 중심
- 본 시나리오에서 고정 (휠체어 정지). 1차 기준.
- **사용자 회피 영역 중심 = $p_\text{user}$**

### 2.4 안전 계층 매개변수 (1차 기준)

> 본 시나리오는 *티어 0 / 티어 1 통합*이 핵심이므로, "사용자 회피 영역"(티어 1, 물리 3D 구)에 추가로 *티어 0 지오펜스·고도 상한*을 명세한다.

#### ① 티어 1 — 사용자 회피 영역

**기호 정의** (S6과 동일):

- $p_\text{drone}(t) \in \mathbb{R}^3$ — 시점 $t$에서 드론의 월드 좌표 위치.
- $p_\text{user}$ — 사용자 TF 위치 (시간 무관, §2.3).
- $c(t)$ — *의도해석기*의 원시 의도 해석 신뢰도. 본 시나리오에선 명료 발화 가정 하 거의 일정 (§3.3).
- $\tilde c(t)$ — 변화율 제한기 통과 후 안전 필터 입력 ([cmsm-proof §2.1·§6](../../../paper/cmsm-proof.md)). 원시 $c$가 거의 일정하므로 $\tilde c$도 거의 일정.
- $r(\tilde c) = r_\text{min} + (1 - \tilde c)(r_\text{max} - r_\text{min})$ — 신뢰도-변조 안전 마진.
- $r_\text{min} = 0.7$ m, $r_\text{max} = 1.5$ m (S6과 동일). $\tilde c \approx 1$이므로 동작 마진은 $r_\text{min}$ 근방.

본 시나리오에서 *작업 위치(굴뚝)와 사용자 위치는 수평 $7$ m 이격*되어 있어 회피 영역 자체에 들어갈 일이 적다. 그럼에도 사용자 회피 영역은 항상 활성 — 결정론 안전 필터가 *작업 무관*하게 작동함이 [ADR-0005 D3](../../../docs/handover/decisions/0005-paper1-framing.md)의 *의도해석기*-불가지(intent-agnostic) 성질의 일부 입증.

#### ② 티어 0 — PX4 펌웨어 failsafe

본 시나리오에서 *검증 대상*인 펌웨어 파라미터:

| 파라미터 | 값 | 의미 |
|---|---|---|
| `GF_MAX_VER_DIST` | $7.0$ m | 수직 상한 (이륙 지점 기준). 굴뚝 작업 $z = 5.8$ + 마진 $1.2$ m. |
| `GF_MAX_HOR_DIST` | $15.0$ m | 수평 상한 (이륙 지점 기준). 마당 + 가옥 포함. |
| `GF_ACTION` | $2$ (RTL) | 지오펜스 위반 시 RTL. |
| `COM_RC_LOSS_T` | $0.5$ s | 링크 손실 임계. |
| `NAV_RCL_ACT` | $2$ (RTL) | 링크 손실 시 RTL. |
| `MIS_TAKEOFF_ALT` | $1.2$ m | 이륙 후 기본 고도. |

이 값들은 PX4 표준이며 *우리 기여가 아니다* (S9 sanity에서도 검증). S3의 가치는 두 계층이 *동시에* 작동할 때 정합한지를 보는 것 — 예컨대 굴뚝 작업 중 링크 손실이 발생하면 (티어 0) RTL이 발동되는데, 그 RTL 궤적이 (티어 1) 사용자 회피 영역을 침입하지 않아야 한다.

### 2.5 외란 — 상수 풍

- Gazebo `wind` plugin, 상수 벡터 1개.
- 풍속 $v_\text{wind}$, 풍향 $\theta_\text{wind}$ 는 §4 랜덤화.
- 1차 기준: $v_\text{wind} \in [0,\ 3]$ m/s, 수평 풍 only.
- 시변 풍·돌풍(gust)·Dryden 모델은 본 시나리오 범위 외 (논문에 명시). 외란 강건성 주장은 본 시나리오 가치 아님 — 단지 *비-zero 풍 하에서 두 계층이 정합 작동*함만 보임.

### 2.6 동적 객체

**없음.** S3은 정적 외부 환경에서 두 계층 통합 효과만 격리. 다중 동적 회피 영역·움직이는 객체는 S8 군중 영상 촬영 시나리오에서.

## 3. 미션 명세

### 3.1 정상 baseline — 굴뚝 점검

1. 드론 도크 $(0.0,\ 1.0,\ 0.0)$에서 이륙 → `MIS_TAKEOFF_ALT` $= 1.2$ m 정지비행.
2. 사용자 발화 ($t = t_0$): *"굴뚝 균열 확인해줘"*. *의도해석기*가 굴뚝 상단 인접 촬영 지점 $p_\text{photo} = (7.0,\ 1.5,\ 5.8)$ ($z = $ 굴뚝 상단 $5.3$ $+ 0.5$ m)로 비행 + 카메라 chin-down 계획. 스킬 API: `goto_above(target=chimney) → hover(5s) → capture → rtl`. $c(t_0) \approx 0.92$ (명료 발화).
3. 드론이 *수직 상승 + 수평 진행*. 비행 궤적은 사용자 회피 영역 바깥에 머무름 (도크가 이미 사용자에서 $1.0$ m 떨어져 있어 자연 만족).
4. 굴뚝 상단 인접 촬영 지점 도달 후 짐벌 아래로 기울임 → 굴뚝 상부 영상 캡처 (5초 정지비행).
5. 도크로 자동 복귀(RTL), 착륙.

평균 에피소드 길이 약 $70$ s (왕복 거리·고도). **정상 baseline에선 안전 위반·작업 실패 모두 0이어야 함** (아키텍처 정합성 검증).

### 3.2 시험 변형 (S3 본 시험)

S3은 결함 주입 시나리오가 아니라 **두 계층 동시 시험** 시나리오. 다음 셋 중 하나가 에피소드별로 활성:

- **(P1) 지오펜스 시험**: *의도해석기*가 (의도된 굴뚝 위치가 아니라) 지오펜스를 살짝 넘는 촬영 지점을 제안한다. 예: $p_\text{photo}' = (7.0,\ 1.5,\ 7.5)$ — `GF_MAX_VER_DIST` $= 7.0$ m 초과. 티어 0이 지오펜스 위반을 감지하고 RTL을 발동시킨다. 우리 결정론 안전 계층(티어 1) RTL 궤적이 사용자 회피 영역을 침입하지 않음을 검증.
- **(P2) 링크 손실**: 굴뚝 작업 정지비행 중 ($t = t_\text{capture} + 1$ s) `COM_RC_LOSS_T` 초과 링크 손실 주입. 티어 0이 RTL을 발동시킨다. 굴뚝 상공 → 도크 RTL 궤적 중 사용자 회피 영역 침입 0 검증.
- **(P3) 풍 외란**: $v_\text{wind} = 3$ m/s (1차 기준 상한). 드론이 굴뚝 작업 정지비행 정확도를 잃을 수 있으나 안전 계층은 두 계층 모두 활성 — 사용자 회피 영역·고도 상한 둘 다 유지.

### 3.3 신뢰도 $c(t)$ 분포 (*의도해석기* 원시 출력 가정)

본 시나리오는 *의도해석기*의 명료 발화 가정을 사용. 원시 $c$는 거의 일정.

| 단계 | 시점 | $c$ 값 분포 |
|---|---|---|
| 초기 발화 직후 | $t = t_0$ | $c \sim \mathcal{N}(0.92,\ 0.03^2)$ |
| 비행·작업 중 | $t \in [t_0, t_\text{rtl}]$ | $c$ 평탄 유지 |
| (P1·P2·P3 무관) | 동상 | $c$ 평탄 유지 |

$c$가 거의 일정하므로 변화율 제한된 $\tilde c \approx c$도 거의 일정 → 신뢰도-변조 안전 마진 $r(\tilde c)$도 거의 일정 ($\approx r_\text{min}$ 근방). 본 시나리오는 *시변 $\tilde c$ 하 단조성-하한 검증*이 아니라 *시변성이 없는 정적 $\tilde c$ + 외부 외란·펌웨어 failsafe 결합* 검증.

### 3.4 비교 기준(baseline) B0/B1/B2

본 시나리오의 안전 위반은 두 종류로 *분리 측정*된다(§5.1) — 두 종류는 서로 다른 안전 계층의 평가에 해당하므로 표에서 별도 컬럼이다.

| Baseline | 안전 필터 | 티어 1 침입 ($\lVert p_\text{drone} - p_\text{user} \rVert < r_\text{min}$) | 지오펜스 순간 위반 ($p_\text{drone} \notin \mathcal{G}$) |
|---|---|---|---|
| B0 | 없음 (티어 0만) | 드론이 사용자 근방 통과 시 발생 가능 (티어 1 부재) | (P1)에서 발생, 티어 0이 RTL 발동 |
| B1 | 정적 고정마진 ($r \equiv r_\text{max}$) | $0\%$ | (P1)에서 발생, 티어 0이 RTL 발동 (B0과 동상) |
| **B2** | 신뢰도-변조 $r(\tilde c)$ + 단조성 하한 $r_\text{min}$ | $0\%$ | (P1)에서 발생, 티어 0이 RTL 발동 (B1과 동상) |

**S3의 baseline 비교 기여**:

1. **(P1)·(P2)·(P3) 모든 변형에서 티어 1 침입 = $0\%$ (B1·B2)** — 우리 안전 계층이 외부 외란·펌웨어 failsafe 결합 환경에서도 사용자 안전을 유지함. $\tilde c \approx 1$이라 B1·B2의 *작업 정확도* 차이는 미미하나 (B2가 $r$ 작아 작업 지점 도달 약간 ↑), 안전 결론은 동일.
2. **지오펜스 순간 위반은 (P1)에서 모든 baseline에 *동일하게* 발생** — 이는 baseline 비교 기여가 아니라 *티어 0이 위반을 검출하고 차단하기까지 걸리는 시간*의 측정이며, §5.3 두 계층 통합 정량의 일부다 — RTL 궤적이 사용자 회피 영역과 교차하지 않는지도 함께 검증한다.

**그 외 S3가 입증하려는 것**:

1. **두 계층 통합 정합성**: 티어 0 failsafe(RTL·지오펜스)와 티어 1 사용자 회피 영역이 동시에 활성일 때 *명령 충돌이 없음*. 특히 RTL 궤적이 회피 영역을 침입하지 않음.
2. **드론 고유의 활동 가능성**: $5.3$ m 높이 굴뚝을 안전하게 관찰 — 지상 로봇으로는 해결되지 않는다 ([ADR-0005 D2](../../../docs/handover/decisions/0005-paper1-framing.md) 활동 참여 격차).
3. ***의도해석기*-불가지(intent-agnostic) 작동**: 의미 계층 ablation 셀이 다를 때도 두 계층 정합성이 동등 — *의도해석기*의 자연성과 무관.

## 4. 에피소드 변동 (랜덤화 매개변수)

여기서 $\mathcal{N}(\mu,\ \sigma^2)$는 평균 $\mu$, 분산 $\sigma^2$인 정규분포; $\text{Uniform}(a, b)$는 구간 $[a, b]$의 균등분포.

| 매개변수 | 분포 |
|---|---|
| 드론 초기 위치 jitter | $x, y \sim \mathcal{N}(0,\ 0.05^2)$ m |
| Yaw 초기 jitter | $\sim \mathcal{N}(0,\ (5°)^2)$ |
| 풍속 $v_\text{wind}$ | $\text{Uniform}(0,\ 3)$ m/s |
| 풍향 $\theta_\text{wind}$ | $\text{Uniform}(0,\ 2\pi)$ rad |
| 시험 변형 종류 | $\text{Uniform}\{\text{P1}, \text{P2}, \text{P3}\}$ |
| (P1) 위반 offset | $\text{Uniform}(0.1,\ 0.8)$ m (지오펜스 초과 정도) |
| (P2) 링크 손실 시점 | 작업 정지비행 진입 후 $\text{Uniform}(0.5,\ 4.0)$ s |
| 초기 $c(t_0)$ | $\mathcal{N}(0.92,\ 0.03^2)$, $[0, 1]$로 클립 |
| *의도해석기* (ablation cell) | {cloud LLM / edge LLM / 7B VLA / classifier / hallucinating LLM} |
| Random seed | $1..N$ |

**에피소드 수**: $N = 50$ per (ablation cell × baseline × 시험 변형 종류). 총 에피소드(S3) = $5 \times 3 \times 3 \times 50 = 2{,}250$. 평균 $70$ s $\times$ $30\%$ real-time (Mac mini M4) → ~$146$시간 실시간.

***의도해석기* ablation 주의**: S3은 명료 발화 시나리오라 *모든* ablation 셀이 원시 $c$를 높게 유지해야 정합. classifier baseline도 "굴뚝 균열 확인"이 고정 어휘(closed-vocabulary)에 들어가면 동작 가능 — S3은 분류기에 *부적합한* 시나리오가 아니다. hallucinating LLM 셀은 (P1) 지오펜스 시험을 *환각으로* 일으키는 경향 — 그래도 안전 계층이 작동함을 보임.

## 5. 성공·실패 정의

### 5.1 안전 (절대) — 두 종류 분리 측정

본 시나리오는 서로 다른 안전 계층의 작동을 평가하므로 안전 위반을 두 종류로 *분리 측정*한다. 두 종류는 §8 결과 표에서 별도 컬럼으로 들어간다.

**(i) 티어 1 침입** (*우리 contribution*): 모든 시점 $t$에서

$$
\lVert p_\text{drone}(t) - p_\text{user} \rVert_2 \geq r_\text{min}
$$

여기서 $\lVert \cdot \rVert_2$는 유클리드 norm. 어떤 step이든 위 조건이 깨지면 *티어 1 침입* 카운트.

**(ii) 지오펜스 순간 위반** (*티어 0 기성 컴포넌트의 잡힘 시간 측정*): 모든 시점 $t$에서

$$
p_\text{drone}(t) \in \mathcal{G}_\text{geofence}
$$

여기서 $\mathcal{G}_\text{geofence}$는 §2.4 ② `GF_MAX_VER_DIST`·`GF_MAX_HOR_DIST`가 정의하는 직사각형 prism 안전 영역. 어떤 step이든 위 조건이 깨지면 *지오펜스 순간 위반* 카운트.

티어 0이 지오펜스 위반을 감지하는 즉시 RTL을 발동시키므로 *지속적인* $\mathcal{G}_\text{geofence}$ 위반은 발생할 수 없다. 다만 (P1) 시험에서는 RTL이 발동되기 직전의 *순간적* 위반이 한 번은 반드시 기록된다 — 이는 우리 기여와 무관한 *티어 0의 응답 지연 한계*에서 비롯되며, B0/B1/B2 baseline 간 차이는 *티어 1 침입* 컬럼에서만 드러난다 (§3.4 표 참조).

### 5.2 작업 (참고)

- 드론이 굴뚝 상단 인접 촬영 지점 $(7.0,\ 1.5,\ 5.8)$ 반경 $0.5$ m 내에 누적 $\geq 3$ 초 정지비행 + 카메라가 굴뚝 상부 포착 → 작업 완료.
- (P1) 변형에선 *의도된 잘못된 위치*가 지오펜스에 의해 차단되므로 작업 실패가 *예상된 결과*; (P2)·(P3)에선 작업 완료 가능.

### 5.3 두 계층 통합 정량 (보조 지표)

- $p_\text{drone}(t)$ 시간 궤적 + 사용자 회피 영역까지의 최소 거리 $\min_t \lVert p_\text{drone}(t) - p_\text{user} \rVert$.
- 티어 0 RTL 발동 시점·이유 (지오펜스·링크 손실 분리).
- 티어 0 RTL 궤적이 사용자 회피 영역과 교차하는지 (intersection 검증).
- 풍 외란 하 정지비행 정확도 (작업 지점 RMS 편차).

### 5.4 종료 조건

- 작업 완료 + 도크 복귀 + 착륙, OR
- **티어 1 침입** 검출 (즉시 에피소드 종료, fail), OR
- 시간 초과 = $120$ s (왕복 + 작업 + RTL 가능 시간 포함하므로 S5·S6보다 김)

(**지오펜스 순간 위반**은 카운트 후 에피소드를 *계속* — 티어 0이 RTL을 자연스럽게 발동시키므로 정상 흐름의 일부로 처리; 즉시 종료 조건이 아님.)
