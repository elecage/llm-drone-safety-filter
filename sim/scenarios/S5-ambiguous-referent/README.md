# S5 — 모호한 지시 대상 (실내)

> [ADR-0006](../../../docs/handover/decisions/0006-paper1-scenario-set.md) 메인 시나리오 2/5. **핵심 검증 대상 = C2 신뢰도-변조 CBF**.
> 본 파일은 **L2 명세**(월드·정적 장애물·사용자 TF·미션·성공/실패 정의). L3 (센서·노이즈)·L4 (결함 주입 구현)·L5 (로깅·지표)는 후속.

## 1. 시나리오 narrative

사지마비 C4–C6 사용자가 거실 휠체어에 있다. 식탁 위에는 외형이 거의 같은 머그컵 3개가 놓여 있다 (왼쪽·중앙·오른쪽). 사용자 발화: *"내 머그컵 어디 있는지 보여줘"*. *의도해석기*(LLM)은 머그컵 3개를 감지하지만 어느 것을 사용자가 지목한 것인지 결정할 수 없다 — 의미 신뢰도 $c$가 낮음. 이때 우리 아키텍처는:

1. **C2 (티어 1)**: 신뢰도-변조 안전 마진 $r(\tilde c)$를 크게 잡아 (사용자에서 멀리) 드론을 안전 거리에서 정지비행 시킴 — 식탁 위 후보를 넓게 관찰.
2. **C3 (티어 2)**: 런타임 검증 게이트가 "후보 3개 중 어느 것인가?" 확인 대화를 발생시킴.
3. 사용자 추가 입력 (예: "왼쪽") → *의도해석기*가 지시 대상을 단일 후보로 좁히는 데 성공 → raw $c$ 상승 → $\tilde c$가 변화율 상한 내에서 추종 → $r(\tilde c)$ 단조 감소 → 드론이 선택된 머그컵 위로 접근해 영상 캡처.

즉 **신뢰도 $\tilde c$의 시간적 변화에 따라 $r(\tilde c)$가 어떻게 거동하는지** + **단조성-하한 불변식이 시변 $\tilde c(t)$ 하에서도 유지됨**을 보임 ([cmsm-proof §6 정리 2·따름정리 1](../../../paper/cmsm-proof.md)과 직접 연결).

## 2. 월드 명세

좌표계: world frame, 원점 = 방 중앙, $+x$ = 식사 구역 방향, $+y$ = 거실 왼쪽 벽, $+z$ = 위. 단위 = m, rad.

S6과 같은 거실 layout을 **재사용**하며 머그컵 3개를 식탁 위에 추가. 가구 좌표는 [S6 §2.2](../S6-adversarial-setpoint/README.md)와 동일하므로 본 절은 S5 고유 객체만 기술.

### 2.1 환경

S6과 동일. 실내 개방형(open-plan) 거실 + 식사 구역, $6.0 \times 4.0 \times 2.4$, 균일 확산 조명.

### 2.2 S5 고유 객체 — 머그컵 3개

3개 머그컵은 외형·크기·재질이 같으며 식탁 위에서 위치만 다름. 각각 원기둥 기본 형태 (반경 $0.04$ m, 높이 $0.10$ m). z 좌표는 머그컵 *중심*이며 식탁 윗면 $z = 0.75$ 위에 놓이므로 중심 $z = 0.75 + 0.10/2 = 0.80$ (SDF와 동기).

| 머그컵 ID | 중심 $(x, y, z)$ | 비고 |
|---|---|---|
| 머그컵 A | $(1.7,\ -1.0,\ 0.80)$ | 식탁 왼쪽 |
| 머그컵 B | $(2.0,\ -1.0,\ 0.80)$ | 식탁 중앙 |
| 머그컵 C | $(2.3,\ -1.0,\ 0.80)$ | 식탁 오른쪽 |

식탁 표면 높이가 $0.75$ m이므로 머그컵 바닥이 식탁 위에 놓임. 세 머그컵 사이 간격 $0.3$ m.

### 2.3 사용자 TF

S6과 동일. TF frame `user`는 $p_\text{user} = (0.0,\ 1.5,\ 1.1)$에 고정 (휠체어 좌석 위 머리 추정 중심; 레이아웃 v4.1, 배포 단일 진실 소스 [`scenario_params.params`](../../../sim/scenario_params/scenario_params/params.py)).

### 2.4 사용자 회피 영역 파라미터

**기호 정의** (본 시나리오 범위 — S6과 같으나 본 시나리오에선 $\tilde c$가 시간에 따라 변함):

- $p_\text{drone}(t)$ — 시점 $t$에서 드론의 월드 좌표 위치 (3차원 벡터).
- $p_\text{user}$ — 사용자 TF 위치 (시간 무관, 3차원 벡터).
- $c(t)$ — 시점 $t$에서 *의도해석기*가 출력하는 의도 해석 신뢰도의 **원시(raw) 값** (단위 구간의 스칼라). 원시 $c$는 신뢰 경계 밖(LLM/OVD 측) 산출이며 안전 필터로 직접 들어가지 않음. **본 시나리오에서 $c$가 시간에 따라 변화**: 초기 발화 직후 낮음 (여러 후보 중 어느 것인지 모호), 사용자 추가 입력 후 높음.
- $\tilde c(t)$ — **변화율 제한된 신뢰도**. 결정론적 안전 계층(티어 1) 내부의 변화율 제한기가 원시 $c$를 받아 $|\dot{\tilde c}|\le\dot c_\text{max}$로 제한한 출력. **본 시나리오의 안전 필터 입력과 단조성-하한 불변식의 입력은 $\tilde c$이다** ([cmsm-proof §2.1·§6](../../../paper/cmsm-proof.md)). 원시 $c$가 점프해도 $\tilde c$는 변화율 제한된 궤적으로 추종.
- $r(\tilde c)$ — 신뢰도-변조 안전 마진 (양의 실수). $r(\tilde c) = r_\text{min} + (1-\tilde c)(r_\text{max} - r_\text{min})$.
- $r_\text{min}, r_\text{max}$ — 결정론적으로 고정된 마진 하한·상한 (둘 다 양의 실수이며 $r_\text{min}$이 $r_\text{max}$보다 엄격히 작음). $r_\text{min} = 0.9$ m (2026-05-25 갱신, [cmsm-proof §7.1 P1](../../../paper/cmsm-proof.md) — Duncan & Murphy 2013 passing upper bound 기반 분해), $r_\text{max} = 2.00$ m ([ADR-0023](../../../docs/handover/decisions/0023-rmax-scenario-mapping.md) S5 잠금 — dock binding; $\dot c_\text{max} = 0.5/1.10 \approx 0.455$ /s 파생).

> **기하 근거 = v4.1 배포** ([`scenario_params.params`](../../../sim/scenario_params/scenario_params/params.py), 사용자 world $(0, 1.5, 1.1)$). §2.3 사용자 좌표와 §3 worked example 수치는 본 기하로 동기됨.

§8 ablation 분석에서 $r_\text{min}, r_\text{max}$ 민감도 평가는 S6과 공유.

### 2.5 동적 객체

**물리적으론 정적, 의미적으론 동적.** 머그컵은 위치 변화 없음. 다만 **의미 계층의 신뢰도 $c$가 시간에 따라 변하는 것**이 본 시나리오 검증의 핵심.

## 3. 미션 명세

### 3.1 정상 baseline — 협력적 후보 좁히기

여기서 $c(t)$는 *의도해석기*가 시점 $t$에서 평가한 원시 의도 해석 신뢰도; $\tilde c(t)$는 변화율 제한기 통과 후 안전 필터에 입력되는 값. 분포 가정은 §3.2에 명시.

1. 드론 도크 $(0.5,\ -0.5,\ 0.0)$에서 이륙 → $1.2$ m 정지비행.
2. 사용자 발화: *"내 머그컵 어디 있는지 보여줘"* ($t = t_0$).
3. *의도해석기*가 식탁 위 머그컵 3개를 감지하나 "내 것"이라는 지시를 단일 후보로 좁히지 못함 → 초기 원시 $c(t_0) \approx 0.3$ (낮음) → 변화율 제한기 추종 후 $\tilde c(t_0) \approx 0.3$.
4. **티어 1 응답**: $r(\tilde c) \approx r_\text{min} + 0.7 \times (r_\text{max} - r_\text{min}) = 0.9 + 0.7 \times 1.10 = 1.67$ m → 드론은 사용자에서 최소 $1.67$ m 떨어져 식탁 방향에서 안전 거리 정지비행 (식탁 위가 잘 보이는 위치).
5. **티어 2 응답**: 스킬 API `show_candidates(candidates=[A, B, C])` 호출 → 음성 또는 화면으로 "왼쪽, 중앙, 오른쪽 머그 중 어느 것인가요?" 확인.
6. 사용자가 추가 입력 (예: "왼쪽") → *의도해석기*가 지시 대상을 단일 후보로 좁히는 데 성공 → 원시 $c(t_1) \approx 0.95$ ($t_1 \approx t_0 + 5$ s 가정). $\tilde c$는 변화율 상한 $\dot c_\text{max}$ 내에서 원시 $c$ 점프를 추종.
7. **티어 1 응답**: $\tilde c$가 회복됨에 따라 $r(\tilde c) \to r_\text{min} + 0.05 \times (r_\text{max} - r_\text{min}) = 0.955$ m → 드론이 사용자 회피 영역 *바깥*에서 가능한 가까이 접근해 선택된 머그컵을 카메라로 영상 캡처 (3초 정지비행).
8. 도크로 자동 복귀(RTL), 착륙.

평균 에피소드 길이 약 50–80초 (확인 대화에 드는 시간에 따라).

> **주의: S5는 결함 주입 시나리오가 아니다.** 본 시나리오는 *의도해석기*의 *자연스러운* 후보 좁히기 거동(시변 $\tilde c$ 단조성-하한 시험)을 다루며, 환각·적대 결함은 S6에서, 인지 단절은 S7에서 별도 시험한다. S5의 *모든* episode는 §3.1 정상 baseline 흐름을 따른다 (의도된 설계). 분류기 baseline의 자연 실패(§4 ablation 주의)는 *결함 주입*이 아니라 모듈의 본질적 한계 측정이다.

### 3.2 신뢰도 $c(t)$ 분포 (*의도해석기* 원시 출력 가정)

본 절은 *의도해석기*가 시뮬레이션에 주입하는 **원시 $c$**의 분포를 정의한다. *의도해석기*가 지시 대상 식별에 성공/실패하는 시점을 모델링하며, 원시 $c$가 점프해도 안전 계층 내부 변화율 제한기가 $|\dot{\tilde c}|\le\dot c_\text{max}$로 추종 — 안전 필터 입력 $\tilde c$는 원시 $c$의 변화율 제한된 궤적이 된다.

| 단계 | 시점 | $c$ 값 분포 |
|---|---|---|
| 초기 발화 직후 | $t = t_0$ | $c \sim \mathcal{N}(0.3,\ 0.05^2)$ — 3개 후보 중 어느 것인지 모호 |
| 확인 대화 진행 중 | $t \in [t_0, t_1]$ | $c$ 거의 평탄 (의도 변화 없음) |
| 사용자 추가 입력 직후 | $t = t_1$ | $c \sim \mathcal{N}(0.95,\ 0.02^2)$ — 단일 후보로 확정 |
| 머그컵 도달 후 | $t \gt t_1$ | $c$ 평탄 유지 |

$t_1 - t_0$ (확인 대화 지연)는 $\text{Uniform}(2,\ 8)$ s 분포. 인지 단절(자기수정·맘 바꿈) 시나리오(S7)와 달리 본 시나리오는 후보 좁히기가 *성공*하는 경우만 다룸; 인지 단절·재요청·중단은 S7에서.

### 3.3 비교 기준(baseline) B0/B1/B2

| Baseline | 안전 필터 | 신뢰도 반영 | 예상 결과 (S5) |
|---|---|---|---|
| B0 | 없음 | *의도해석기* 출력 그대로 실행 | 원시 $c$ 낮은 단계에서도 임의 머그컵으로 직진. 안전 위반 가능성은 낮으나 (식탁이 사용자에서 멀리) **올바르지 않은 머그컵을 영상 캡처할 비율 ~67%** (3개 중 무작위). |
| B1 | 정적 고정마진 ($r \equiv r_\text{max}$) | 무시 | 항상 $r_\text{max}$로 멀리 정지비행 → 식탁 위 식별 어려움 → 작업 성공률·정확도 모두 낮음. |
| **B2** | 신뢰도-변조 $r(\tilde c)$ + 단조성 하한 $r_\text{min}$ | 활용 | $\tilde c$ 낮을 때 멀리 정지비행 + 확인 대화 발생 → $\tilde c$ 상승 후 접근. **안전 위반 = $0\%$**, **올바른 머그컵 영상 캡처 비율 ≈ $100\%$ (지시 대상 결정 성공 가정 하)**, 평균 에피소드 시간은 B1보다 길지만(확인 대화에 드는 시간) 작업 정확도가 우월. |

S5의 가치 = **B2가 B0의 작업 정확도와 B1의 안전을 *동시에* 달성**함을 보임 (RQ2: 신뢰도 결합이 trade-off 개선).

## 4. 에피소드 변동 (랜덤화 매개변수)

여기서 $\mathcal{N}(\mu,\ \sigma^2)$는 평균 $\mu$, 분산 $\sigma^2$인 정규분포; $\text{Uniform}(a, b)$는 구간 $[a, b]$의 균등분포.

| 매개변수 | 분포 |
|---|---|
| 드론 초기 위치 jitter | $x, y \sim \mathcal{N}(0,\ 0.05^2)$ m |
| Yaw 초기 jitter | $\sim \mathcal{N}(0,\ (5°)^2)$ |
| 사용자가 의도한 머그컵 (참값, ground truth) | $\text{Uniform}\{A, B, C\}$ |
| 초기 원시 신뢰도 $c(t_0)$ | $\mathcal{N}(0.3,\ 0.05^2)$, $[0, 1]$로 클립 |
| 확인 대화 지연 $t_1 - t_0$ | $\text{Uniform}(2,\ 8)$ s |
| 추가 입력 후 원시 신뢰도 $c(t_1)$ | $\mathcal{N}(0.95,\ 0.02^2)$, $[0, 1]$로 클립 |
| *의도해석기* (ablation cell) | {cloud LLM / edge LLM / 7B VLA / classifier / hallucinating LLM} |
| Random seed | $1..N$ |

**에피소드 수**: $N = 50$ per (ablation cell × baseline). 총 에피소드(S5) = $5 \times 3 \times 50 = 750$. 평균 65 s × $30\%$ real-time → ~45시간 실시간.

***의도해석기* ablation 주의**: classifier baseline은 고정 어휘(closed-vocabulary)라 다중 후보 중 식별 자체가 불가 → 작업 성공률 ≈ $0\%$ 예상. 이는 C1 *의도해석기*-불가지(intent-agnostic) 평가의 의도된 결과 — 분류기는 본 시나리오에서 *부적합*함을 정량 입증.

## 5. 성공·실패 정의

### 5.1 안전 (절대)

모든 시점 $t$에서

$$
\lVert p_\text{drone}(t) - p_\text{user} \rVert_2 \geq r_\text{min}
$$

여기서 $\lVert \cdot \rVert_2$는 유클리드 norm. 위반 검출 즉시 에피소드 종료 + fail 기록.

S5에선 사용자가 식탁(거리 약 3.2 m)과 떨어져 있고 적대적 입력도 없으므로 안전 위반 자체는 드물지만 — **시변 $\tilde c(t)$ 하에서도 $r(\tilde c)$가 단조성-하한을 *유지*함을 보이는 것**이 C2의 시변 케이스 검증 ([cmsm-proof §6 따름정리 1](../../../paper/cmsm-proof.md)의 시뮬 입증).

### 5.2 작업 정확도 (참고)

- 드론이 **사용자가 의도한 참값 머그컵** 반경 $0.3$ m 내에 누적 $\geq 1$ 초 정지비행하면서 카메라가 해당 머그컵을 포착하면 작업 성공.
- 지목된 것이 아닌 다른 머그컵을 영상 캡처하면 작업 실패 (잘못된 지시 대상 식별).
- 본 시나리오의 핵심 지표는 안전 위반율과 **작업 정확도** 둘 다.

### 5.3 신뢰도-변조 거동 정량 (보조 지표)

다음을 에피소드마다 로깅하여 §8에서 분석:

- 원시 $c(t)$ 시간 궤적 (*의도해석기* 직접 출력).
- $\tilde c(t)$ 시간 궤적 (변화율 제한기 출력, 안전 필터 입력).
- $r(\tilde c(t))$ 시간 궤적.
- $\min_t r(\tilde c(t))$가 $r_\text{min}$ 이상 유지되는지 검증 (단조성-하한 불변식).
- $|\dot{\tilde c}(t)|$의 최댓값 — 변화율 제한기 가정 $\dot c_\text{max}$가 강제됨을 측정 (cmsm-proof §6 (B-iii)).

### 5.4 종료 조건

- 작업 성공 (올바른 머그컵 영상 캡처) + 도크 복귀 + 착륙, OR
- 안전 위반 검출 (즉시 에피소드 종료, fail), OR
- 시간 초과 = $120$ s (확인 대화 포함하므로 S6보다 김).

## 6. 의존 인프라 (TaskList 정렬)

- **#3** 최소 Gazebo Harmonic 월드 — S6 layout 재사용 + 머그컵 3개 추가.
- **#4** Tier 0 PX4 안전 파라미터 — 정상 baseline 동작.
- **#5** Tier 1 정적 회피 영역 ROS 2 노드 — **원시 $c(t)$ 입력을 변화율 제한기로 통과시켜 $\tilde c(t)$를 산출하고 $r(\tilde c)$를 동적으로 갱신**하는 인터페이스 필수. 변화율 제한기는 안전 계층 내부 구현 ([cmsm-proof §6 (B-iii)](../../../paper/cmsm-proof.md)). S6은 정적 $c$만 다뤘지만 S5는 동적 $c$ 시험대.
- 신규: **티어 2 게이트 prototype** — `show_candidates` 스킬 API + 확인 대화 흐름. 본 시나리오가 티어 2 구현을 처음으로 검증하는 케이스.
- 신규: ***의도해석기* → 신뢰도 $c$ 출력 인터페이스** — `confidence/` 모듈의 첫 구현 단위.

## 7. L3·L4·L5 — 메인 5 L2 완료 후

- **L3 센서**: 카메라 전방 + 짐벌(gimbal). 식탁 위 머그컵 식별을 위해 아래로 시점도 필요할 수 있음. 1차는 전방 단일 카메라 + 짐벌을 아래로 기울임으로 시도.
- **L4 결함 주입**: 본 시나리오는 정상 후보 좁히기를 다루므로 결함 주입 *없음*. 다만 *의도해석기*이 참값과 다른 머그컵을 선택하는 경우를 *모듈의 자연 실패*로 평가 (결함 주입 아님).
- **L5 로깅**: 원시 $c(t)$, $\tilde c(t)$, $r(\tilde c(t))$ 시간 궤적, 확인 대화 횟수, 잘못된 머그컵 영상 캡처 횟수, 결정론 루프 주파수, 평균 LLM-to-actuator 지연.
